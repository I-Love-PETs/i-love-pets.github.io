# v0.4 Release Notes

!!! info "Release status"
    Status: In progress
    Last updated: 2026-06-05
    Scope: Planned v0.4 improvements for decision-ready PET guidance.

v0.4 focuses on making I Love PETs more usable as a decision kit: clearer use cases, reusable benchmark scorecards, stronger architecture review, and better evidence discipline.

## Planned Improvements

| Area | Intended improvement |
| --- | --- |
| Use cases | Turn domain summaries into playbooks with scenarios, PET stacks, non-use guidance, failure modes, and measurements. |
| Benchmarks | Add scorecards for private RAG, private inference, cross-silo FL, synthetic data release, and federated/MPC analytics. |
| Architectures | Add trust-boundary reviews, assumptions, non-protections, and evaluation checklists. |
| Evidence | Strengthen claim states, dated review workflow, and claim-register backlog. |
| Tools | Add a framework for evaluating PET tools by fit, threat model, evidence, operations, and first benchmark. |
| Start Here | Improve onboarding with wrong-assumption checks, routing, and practical glossary examples. |
| Site quality | Add internal-link checking and keep navigation aligned with new guidance. |

## Release Checklist

- `mkdocs build` passes.
- Internal-link check passes.
- New navigation entries point to existing pages.
- Decision pages include at least one concrete table, checklist, worked example, or scorecard.
- No page claims a PET solves privacy without naming assumptions and outputs.
- Open v0.4 PRs are merged in an order that avoids stale links.

## Merge Order

Recommended order:

1. Use-case playbooks.
2. Benchmark scorecards.
3. Architecture trust boundaries.
4. Evidence and claim register.
5. Tool evaluation framework.
6. Start Here polish.
7. Site polish and release notes.
